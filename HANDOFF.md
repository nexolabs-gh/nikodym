## Callejones sin salida / no reintentar
- **CI ≠ gates locales (B4.3, 2026-06-25):** los gates locales (1 OS, 1 Python, sin `--all-extras`) NO detectan fallos de **Windows**, **Python 3.13** ni del **closure completo de licencias (Linux)**. Un commit con gates locales 100% verdes puede romper el CI. → Tras pushear algo con CI, **verificar el run real (`gh run view`)** antes de cerrar el ítem.
- **El gate anti-copyleft NO debe fallar ante licencias no-copyleft no listadas (B4.3):** `nvidia-nccl-cu12` = `LicenseRef-NVIDIA-Proprietary` (no copyleft, wheel **Linux-only**) rompía el build por estar fuera de la allowlist permisiva. El gate debe bloquear SOLO copyleft real (GPL/LGPL/AGPL); lo demás no debe romper. Validar el checker en Linux (ahí está el closure CUDA).
- **Hashes golden de archivos sensibles a EOL (B4.3):** crear archivos de prueba con `write_text` mete `\r\n` en Windows → sha256 distinto → golden roto. Usar `write_bytes`/`newline=""` para contenido que se va a hashear.
- **`Study.save` no es Windows-safe (B4.3):** el swap atómico (`core/study.py:372`, `os.replace` `estudio`→`.estudio.old.xxx`) lanza `PermissionError [WinError 5]` en Windows. Pendiente de B4.4 (si Cami aprueba soporte Windows).
- **`ortools` sin wheel cp313 (B4.3):** `ortools==9.10.4067` (vía optbinning/scoring) solo trae cp311/cp312 → bloquea Python 3.13 en cualquier OS. Pendiente de B4.5.
- **NO `from nikodym.data...` (ni `audit`/`governance`/`tracking`/`mlflow`) a nivel de módulo en `core/`** — rompe el núcleo liviano (gate `import nikodym.core` sin data/pandera/pyarrow/pandas/tracking/mlflow). El registro de dominios se puebla por **import perezoso DENTRO de método** (`importlib.import_module` en `Study._ensure_domain_registered`); las secciones `audit`/`governance`/`tracking` son `Any` en runtime + hook diferido poblado al importar el paquete. NUNCA el import a nivel módulo en `core/`.
- **NO comparar valor vs columna de dtype incompatible sin guardia** (B2b.4 + B2c.1) — `series.eq(x)`/`.lt(x)` cross-dtype emite `FutureWarning` → rompe con `filterwarnings=error`. Filtrar por dtype ANTES (`_is_comparable` en special, `_validate_value_compatible` en target) y `comparison.fillna(False).astype(bool)`.
- **NO usar `model_rebuild()` para narrar `NikodymConfig.data`** — no funciona en Pydantic 2.13. El hook+validador es la vía.
- **NO `Field(strict=True)` sobre uniones** en Pydantic 2.13 — usar el modo smart (default).
- **NO defaults posicionales en `Field(...)`** si el modelo es target de `default_factory` — keyword `default=`.

- **NO `.values.tobytes()` para hashes reproducibles** (B2b.3) — es endian-NATIVO. Forzar little-endian: `hash_pandas_object(...).to_numpy(dtype="<u8", copy=True).tobytes()`. Normalizar `-0.0→0.0` y pinear `hash_key`/`encoding`/`categorize`.
