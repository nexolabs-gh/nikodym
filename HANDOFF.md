## Callejones sin salida / no reintentar
- **CI ≠ gates locales (B4.3, 2026-06-25):** los gates locales (1 OS, 1 Python, sin `--all-extras`) NO detectan fallos de **Windows**, **Python 3.13** ni del **closure completo de licencias (Linux)**. Un commit con gates locales 100% verdes puede romper el CI. → Tras pushear algo con CI, **verificar el run real (`gh run view`)** antes de cerrar el ítem. _(En B4.3 v2 se cumplió: CI verde confirmado.)_
- **El gate anti-copyleft NO debe fallar ante licencias no-copyleft no listadas (B4.3) — RESUELTO en v2:** el checker (`build`→"Verify runtime licenses") ahora bloquea SOLO copyleft real (regex GPL/LGPL/AGPL + "GNU … General Public License"); propietarias (`LicenseRef-*`/`nvidia-*`) y metadata ausente pasan con warning. NO volver a la allowlist permisiva por-paquete (frágil).
- **Hashes golden de archivos sensibles a EOL (B4.3) — MITIGADO:** crear archivos de prueba con `write_text` mete `\r\n` en Windows → sha256 distinto. Usar `write_bytes`/`newline=""` para contenido que se va a hashear (aplicado en `test_audit_environment_hashing.py`). Revisar este patrón en tests nuevos que hasheen archivos.
- **`Study.save` no es Windows-safe (B4.3):** el swap atómico (`core/study.py:372`, `os.replace` `estudio`→`.estudio.old.xxx`) lanza `PermissionError [WinError 5]` en Windows. Pendiente de B4.4 (si Cami aprueba soporte Windows).

- **NO `from nikodym.data...` (ni `audit`/`governance`/`tracking`/`mlflow`) a nivel de módulo en `core/`** — rompe el núcleo liviano (gate `import nikodym.core` sin data/pandera/pyarrow/pandas/tracking/mlflow).
